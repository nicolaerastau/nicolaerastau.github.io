B3&B4-reflektioner

Arbetssätt:

Vi började med att nämna vilken typ av företag vi ville vara. Vårt företag ska vara en webbyrå och ska skapa i olika tjänster till våra kunder. Vi har gjort förslag till namn, logotyp som vi sedan röstade på. Vi har justerat lite gran logotypen med lite text och färger (alla kom med förslag). Nästa steg var att välja MOCKUP och vem ska skapa Git master. Vi har fem sektioner och vi har delat dem varsitt och vi startade koda HTML och CSS. Vi har använt mest Slack. För B4 vi har använt Bootstrap som var för mig lite svårt eftersom boxarna funkade inte som de skulle.



Reflektioner:

På distans är det svårt att samla folk på bestämda tider för att nå målet. Vi har haft lite problem för att Thomas har lämnat gruppen med de som har mer erfarenhet har tagit över uppgifterna och i slutet har de blivit bra. För mig som nybörjare i branschen det har varit en utmanning att jobba i grupp och jag hat fått hjälp från de andra med högre kunskapsnivå. Tyvärr så kunde vi inte ha flera träffar på Slack för att förstå bättre varandra.

Skiss B2_Uppgift

Jag har tänkt att göra en sida med lite information om 3 fotbal lag från Europa. På första sidan det kommer att finnas en bild från varje lag och lite text. Titeln ska finnas längst upp på sidan och under den ska finnas en knapp som ska leda till en annan sida med artiklar om dem tre lagar. Under varje artikel ska finnas en knapp för att gå till oficiella sidor.Sidan ska ha ett logo vilken ska vara på höger sidan.

Kod

Titeln ska vara ganska stort längst upp ock ska ha shadow på röd background. Hela sidan ska ha en orange background. texten ska vara med svart färg. Kontakt rubriken ska bli med vit text ock blå background. För att bilderna ska vara tydliga jag ska lägga border på varje bild. Information sidan ska ha titel med shadow på röd background. Artiklarna ska ha grå background och namnet på varje lag ska bli svart på blå background.

Webben om 5 år

Först i framtiden ska höja hastigheten på internet och det ska bli billigare och mera ska använda det.Web sidorna ska vara mer komplexa. Vi ska vinna lite mer tid när vi ska leta informationen på web till exempel från myndigheter, om fritid,om jobb,om resor osv. En viktigt aspekt tror jag är att en web sida måste inte vara fulla med aggresiva reklamer. Man är trött när man letar efter någonting och hamnar på olika reklam och det måste separera dem på någon sätt. En annan aspekt är vilken information som vi får? En riktigt information eller någonting annat? Jag mennar det måste finnas mer transparens till ex. från myndigheter och statliga organisationer. Tekniken kommer att utveckla med mera interaktiva functioner.Touch screen ska bli mer användbar. I fem år jag tror inte att det ska hända någontig stort ( som Edison hade uppfunnit glödlampa till ex.) men det kommer sakta, "step by step". Sociala medier är nu en del av vårt liv men vad ska vi göra med våran privata liv? Jag personligt tror att det finns tankar, saker och evenemanger som är privata och man kan inte dela med alla! Ibland man vill dela men ändå? PS Vi ska träffa efter 5 år för att ta reda på vem hade rätt!!!

Frontend-utvecklare

A. I början jag ska skriva lite information om yrket. Front-end Developer I modern datateknik associeras ofta front-end med webbaserade mjukvarumoduler baserade på HTML, CSS och JavaScript, där utvecklarna av dessa delar kallas webbutvecklare eller webbdesigners. En del tjänster är mer tekniska medans andra tjänster är mer åt webbdesign-hållet. En teknisk Front-end Developer. En tekniskt fokuserad front-end developer är specialiserad på tekniker som hjälper till att skapa webbgränssnitt. Det kan innebära: HTML5; CSS3; JavaScript – inte bara jQuer ; Webbprestanda och webboptimering; MVC-lösningar i ex PHP, Python, Ruby, ASP.NET eller liknande; Utvecklingsverktyg som effektiviserar utveckling; Git, Mercurial, Subversion; Preprocessors för JavaScript och CSS; Deployscript Front-end Developer med webbdesign-fokus: En front-end developer som är mer åt designhållet är mer specialiserad på att formge webbgränssnitt och har ett lite annat skill-set än den tekniska front-end-utvecklaren. En front-end developer med webbdesign-inriktning har ofta följande saker i sin verktygslåda: HTML5; CSS3; jQuery och liknande JavaScript-bibliotek; Grafiskt kunnig i att formge webbiga gränssnitt; Kan skapa interaktionsdesign-dokument (wireframes); Optimering av bilder, css och javascript-kod med avseende på webbprestanda; Kan skapa webbramverk med återanvändbara komponenter och templates (ex Bootstrap); Duktig på designmjukvara som Photoshop. Det finns front-end developers som kombinerar dessa skillset, men ofta så har olika front-end developers olika fokus. En bra sak att tänka på när man skapar sin rollbeskrivning för sin jobbannons är att tänka på vilken typ av utvecklare man letar efter. HTML (förkortning för HyperText Markup Language) är ett märkspråk för hypertext och utgör tillsammans med TCP/IP och HTTP den grundläggande standarden för WWW (World Wide Web webben); webbsidor skrivs i allmänhet som HTML och överförs över Internet med HTTP. CSS-Cascading Style Sheets ( på svenska stilmall) är ett språk som beskriver presentationsstilen för ett strukturerat dokument som till exempel typsnitt, textstorlek och färg. Tekniken är ett sätt att anpassa dokumentet med hänsyn till datortyp, skärmupplösning, färgdjup och installerade typsnitt. Javascript är ett prototyp-baserat skriptspråk som är dynamiskt, svagt typat och hanterar funktioner som första-klass-objekt. Javascript används främst på klientsidan i webbtillämpningar, det vill säga exekveras i en webbläsares javascript-motor. Då Javascript används i webbläsare arbetar det mot ett gränssnitt som kallas Document Object Model (DOM).

C. Framtiden

Enligt Mattias Petter Johansson, Software Engineer at Spotify: “With ES6 , JavaScript is getting a ton of new features targeted towards making life easier for people building transpilers, i.e. languages that compile down to JavaScript, like CoffeeScript and C++. With asm.js and WebGL, it's actually now possible to run real games that run in the browser without plugins - Brendan Eich has done some very impressive demos with Unreal Tournament running in the browser without plugins. It's slowly but surely becoming the Assembler of the web. JavaScript is even bringing functional programming to the mainstream. JavaScript programmers are using monads on a daily basis without without knowing it. The little functional reactive programming library Bacon.js alone has more stars on Github than the Haskell compiler has. This is tremendously important, because with CPUs now nearing the theoretical speed limit per core, multi-core is the way forward, which will drive the industry towards functional programming which eases the complexity of concurrent programming quite abit. No matter your opinion of JavaScript, it would be hazardous to your career as a programmer not to know it. It's likely that JavaScript will not only maintain, but extend it's hold of the software industry during the coming years.” Citat från https://www.quora.com/Whats-the-future-of-JavaScript
